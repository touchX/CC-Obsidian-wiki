## 💡 优化建议

### 高优先级（立即执行）

#### 1. 更新 Wiki Lint Report
```bash
cd wiki && ../.claude/skills/wiki-lint/wiki-lint.sh > ../WIKI-LINT-REPORT.md 2>&1
```
**收益：** 恢复健康监控可见性

#### 2. 清理 raw/notes/ 归档
```bash
# 选项 A：归档到 archive/resources/notes/
mv raw/notes/*.md archive/resources/notes/

# 选项 B：直接摄取到 Wiki（如果内容有价值）
# 使用 docs-ingest skill 处理
```
**收益：** 恢复三层架构完整性

#### 3. 修复 Wiki Log 编码
```bash
# 检测文件编码
file wiki/log.md

# 转换为 UTF-8
iconv -f GB18030 -t UTF-8 wiki/log.md > wiki/log-fixed.md
mv wiki/log-fixed.md wiki/log.md
```
**收益：** 恢复日志可追溯性

#### 4. 修复 wiki-lint.sh 错误处理
```bash
# 添加详细错误输出
cd wiki && bash -x ../.claude/skills/wiki-lint/wiki-lint.sh 2>&1 | tee lint-debug.log
```
**收益：** 可靠的健康检查

### 中优先级（本周执行）

#### 5. 增强 Memory Hub 使用
**当前：** 会话结束前询问是否存储  
**优化：** 关键里程碑自动存储
- 完成 workflow 后自动存储经验
- 重大问题解决后存储根因
- 新技能学习后存储用法

**实现：**
```bash
# 在 self-correction.md 中添加
## 自动存储触发条件
- 完成完整 workflow (brainstorming → planning → execution)
- 解决 complex bug (systematic-debugging >30 分钟)
- 学习新 tool/skill 的首次成功使用
```

#### 6. Wiki 索引优化
**问题：** wiki/index.md 和 index-auto.md 重复  
**优化：** 统一为单一索引，使用 Dataview 自动生成

```markdown
```dataview
TABLE without id
  file.link as "页面",
  description as "描述",
  type as "类型",
  tags as "标签"
FROM "wiki"
WHERE file.path != "wiki/index.md"
SORT type, file.name
```
**收益：** 维护成本降低，一致性提高

#### 7. 技能工作流自动化
**问题：** Wiki 归档需 8 个手动步骤  
**优化：** 创建一键归档脚本

```bash
# .claude/scripts/quick-archive.sh
#!/bin/bash
file=$1
obsidian create path="wiki/$(basename $file .md).md" content="$(cat $file)"
# 自动设置属性...
# 自动更新 log...
# 自动归档...
```

**收益：** 归档时间从 5 分钟降至 30 秒
