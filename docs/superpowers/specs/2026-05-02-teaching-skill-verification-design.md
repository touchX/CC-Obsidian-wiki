# 教学 Skill 强制性 Wiki-First 验证机制设计

> **日期**：2026-05-02
> **类型**：设计文档
> **状态**：待用户评审

## 1. 问题陈述

**现象**：
- `mentor-ai-programming` skill 在教学中，不经过查询权威文档或本项目 Wiki 就产生教程
- 产生严重的幻觉内容，给学习者带来错误导向
- 违反 Wiki-First 原则

**根因**：
- 教学 Skill 缺乏 Pre-Teaching Verification Gate
- 即使 Skill 描述了数据源（Wiki），也没有技术强制要求在输出前必须验证
- AI 的自信输出 vs 真实准确性之间存在系统性偏差

**影响范围**：
- `mentor-ai-programming` 及所有教学类 Skills
- 需要建立跨所有教学 Skill 的强制性 Wiki-First 验证机制

---

## 2. 设计目标

| 目标 | 描述 |
|------|------|
| **权威性** | 事实层内容必须来自 Wiki，确保绝对准确 |
| **创造性** | 洞察层允许 AI 发挥，但需透明标注 |
| **进化性** | 不阻塞 AI 持续学习和归纳新知识的能力 |
| **可追溯性** | 所有未验证输出自动记录，支持持续改进 |

---

## 3. 分层置信度模型

### 3.1 三层架构

```
教学内容
│
├── 📌 事实层（Facts）— 必须来自 Wiki
│   ├── 定义：语法、用法、配置项、命令参数
│   ├── 要求：必须引用具体 Wiki 页面和行号
│   └── 违规：严格禁止，直接拒绝输出
│
├── 💡 洞察层（Insights）— AI 可生成，需标注
│   ├── 定义：应用场景联想、知识点综合、类比解释
│   ├── 要求：必须标注"AI 洞察"及置信度
│   └── 示例："基于对 hooks 工作机制的理解，这个场景..."
│
└── 🔮 推测层（Speculations）— 高度不确定，需强调
    ├── 定义：预测未来功能、未经证实的联想
    ├── 要求：必须标注"推测内容，官方未确认"
    └── 示例："预计 v2.0 可能支持..."
```

### 3.2 各层处理规则

| 层级 | Wiki 有相关内容 | Wiki 无相关内容 |
|------|-----------------|-----------------|
| **事实层** | ✅ 基于 Wiki 输出 | ❌ 拒绝输出，必须标注"未验证" |
| **洞察层** | ✅ 基于 Wiki + AI 综合 | ✅ AI 生成，但必须标注"未经验证" |
| **推测层** | ✅ 基于趋势分析 | ⚠️ 允许，但必须加强标注 |

### 3.3 标注模板

**洞察层标注**：
```markdown
> [!tip] AI 洞察
> 💡 此分析基于对 [[claude-commands]] 和 [[hooks]] 的综合理解，
> 仅供参考，建议实践中验证。
```

**推测层标注**：
```markdown
> [!warning] 推测内容
> ⚠️ 此为基于当前趋势的推测，官方尚未确认，谨慎参考。
```

**未验证内容标注**：
```markdown
> [!caution] 未经验证
> ⚠️ 此内容基于训练数据，Wiki 中暂无相关记录，
> 准确性无法保证，请以官方文档为准。
```

## 4. 技术实现方案

### 4.1 方案选择

| 决策点 | 选择 | 理由 |
|--------|------|------|
| **验证位置** | Skill 内部拦截 | 改动最小，行为内聚 |
| **验证方式** | 强制调用 wiki-query | 技术强制，复用成熟组件 |
| **无结果行为** | 分层处理（B1 变体） | 保证权威性的同时保护创造力 |
| **错误追踪** | 自动记录（C1） | 建立持续改进机制 |

### 4.2 标准工作流

```
用户请求教学
    │
    ▼
┌─────────────────────────────────────────┐
│ 【强制门】wiki-query 搜索相关 Wiki 页面 │
└─────────────────────────────────────────┘
    │
    ├── 找到相关内容
    │   │
    │   ▼
    │   读取 Wiki 页面内容
    │   │
    │   ▼
    │   【分层输出】
    │   ├── 事实层 → 基于 Wiki，标注来源
    │   ├── 洞察层 → AI 综合，标注"💡 AI 洞察"
    │   └── 推测层 → 谨慎推测，标注"⚠️ 推测"
    │
    └── 未找到相关内容
        │
        ▼
        【分层处理】
        ├── 事实层请求 → 拒绝，明确说明"Wiki 无相关内容"
        ├── 洞察层内容 → 允许输出，标注"⚠️ 未经验证"
        └── 推测层内容 → 允许输出，加强警告标注
```

### 4.3 Skill 集成要求

所有教学类 Skill（`mentor-ai-programming` 及其调用链）在 SKILL.md 头部添加：

```markdown
## Pre-Teaching Verification Gate

**触发条件**：每次准备输出教学内容前

**执行流程**：
1. 【强制】调用 wiki-query 搜索相关 Wiki 页面
2. 【判断】根据搜索结果决定输出层
3. 【标注】所有内容必须标注来源或置信度
4. 【记录】未验证内容自动记录到 error-tracking

**禁止行为**：
- 未经 wiki-query 验证直接输出事实层内容
- 隐藏内容的置信度
- 将洞察/推测当作事实输出
```

## 5. 错误追踪集成

### 5.1 自动记录触发条件

当出现以下情况时，自动记录到 `error-tracking.md`：

| 触发场景 | 记录类型 | 数据点 |
|----------|----------|--------|
| Wiki 无相关内容但输出了洞察层 | warning | 主题、AI 置信度、用户反馈 |
| 推测层内容被用户质疑 | error | 推测内容、依据、质疑点 |
| 事实层被用户发现错误 | critical | 错误内容、正确内容、来源对比 |

### 5.2 记录格式

```yaml
error_id: AUTO-<timestamp>
date: <current-date>
severity: warning | error | critical
source: auto-detected
category: unverified-content
topic: <teaching-topic>
content: <original-ai-output>
confidence_label: "⚠️ 未经验证" | "💡 AI 洞察" | "⚠️ 推测"
user_feedback: <if-available>
improvement: <suggestion>
```
