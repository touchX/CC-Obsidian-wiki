**日期**: 2026-04-27

## 1. 设计目标

设计一个具有多年编程和教育经验的 AI 编程教学 Agent，根据本 Wiki 中的 Claude Code 最佳实践资料，由浅入深地创建系统课程。

### 1.1 核心目标

- 面向**进阶者**（有编程经验，想深入掌握 Claude Code）
- 采用**阶段闯关 + 模块自由**的学习形式
- 使用**代码审查式**评估学习效果
- 支持**快速模式/学习模式**切换
- 混合调用**内置课程 + Wiki 动态内容**

| 阶段 | 名称 | 描述 | 评估方式 |

## 4. 交互模式

### 4.1 快速模式 vs 学习模式

| 模式 | 特点 | 使用场景 |
|------|------|----------|
| **快速模式** | 直接给出答案和解释 | 时间紧迫、需要快速解决问题 |
| **学习模式** | Socratic 提问引导 | 深入理解、培养独立思考 |

### 4.2 学习模式核心原则（借鉴 mentoring-juniors）

- **从不直接给答案** — 引导学习者自己发现解决方案
- **分阶段提示** — 轻度→中度→强提示，根据阻塞程度递进
- **错误转化** — 将错误视为学习机会，而非失败
- **事后复盘** — 🔴 高压交付后必须复盘学习

### 4.3 Socratic 教学流程

```
问题情境 → 引导提问 → 学习者思考 → 提示递进 → 总结复盘
    ↑_______________________________________________|
```

## 5. 评估系统

### 5.1 代码审查式评估

Agent 扮演 Code Reviewer 角色：

| 维度 | 检查项 |
|------|--------|
| **功能性** | 代码是否正确实现需求？边界情况？ |
| **最佳实践** | 是否遵循 Claude Code 最佳实践？ |
| **可读性** | 代码是否清晰易懂？ |
| **安全性** | 是否有安全隐患？ |

### 5.2 挑战任务设计

每个模块包含：
- **L1**: 3-5 个基础任务
- **L2**: 3-5 个进阶任务
- **L3**: 1-2 个综合项目

## 7. Skill 接口设计

### 7.1 命令格式

```
/mentor-ai-programming [模式] [模块] [任务] [选项]
```

### 7.2 参数说明

| 参数 | 可选值 | 说明 |
|------|--------|------|
| `模式` | `fast` / `learn` | 快速模式 / 学习模式 |
| `模块` | `commands` / `hooks` / `subagents` / `workflows` / `teams` / `all` | 教学模块 |
| `任务` | `status` / `start` / `challenge` / `review` | 查看状态 / 开始 / 挑战 / 代码审查 |
| `选项` | `--level=1-3` / `--reset` | 指定难度 / 重置进度 |

### 7.3 使用示例

```bash
# 开始 Commands 模块学习
/mentor-ai-programming learn commands start

# 查看当前进度
/mentor-ai-programming status

# 开始 L2 挑战
/mentor-ai-programming learn workflows challenge --level=2

# 提交代码审查
/mentor-ai-programming fast hooks review --file=hook-config.ts
```
