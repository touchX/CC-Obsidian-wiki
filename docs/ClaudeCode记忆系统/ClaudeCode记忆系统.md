## 一、设计思想

### 1.1 问题背景

在长时间使用 Claude Code 进行开发的过程中，我们面临以下挑战：

- **信息分散**: 工作记忆、会话记忆、长期知识分属不同系统，难以统一检索
- **经验流失**: 跨会话积累的专业知识容易被遗忘
- **重复劳动**: 每次开始新会话都需要重新探索已知内容
- **质量不一**: AI 回答质量不稳定，偶尔出现幻觉内容

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| **分层存储** | 不同类型的记忆使用最适合的工具 |
| **语义优先** | memU 提供语义搜索，claude-mem 提供时间序列检索 |
| **自动与手动结合** | 自动化采集高价值信号，人工审核关键知识 |
| **四层协作** | 每层各司其职，通过 Memory Hub 统一入口协调 |

## 三、工具与实现

### 3.1 memU (MCP Server)

**核心组件**:
- Docker 容器中的 Python 应用
- PostgreSQL 数据库存储
- REST API (`localhost:8000`) + MCP 接口双重访问

**关键功能**:
```bash
# 存储记忆
memU_store(content="...", category="tech")

# 语义检索
memU_retrieve(query="...")

# 类别统计
memU_categories()

# 系统状态
memU_stats()
```

**配置信息**:
- 端口: 8000
- 数据库: PostgreSQL
- 路径: `~/.claude/memu/` 或 Docker 容器内 `/app/`

### 3.2 Claude-Mem Skill

**核心功能**:
- `mem-search`: 近期记忆搜索
- `list-corpora`: 查看所有知识库
- `timeline-report`: 时间线报告
- `smart-explore`: 智能探索

**特点**:
- 基于时间维度的记忆检索
- 支持会话间上下文
- 与 claude-mem MCP search 协同工作

### 3.3 Memory Hub Skill

**设计目标**: 成为记忆系统的统一入口

**核心文件**:
```
~/.claude/skills/memory-hub/
├── SKILL.md           # 主技能定义
├── suggest.js         # 智能建议
├── docs/
│   └── user-guide.md  # 用户指南
└── scripts/
    └── sync.sh       # memU 与 claude-mem 同步脚本
```

**功能矩阵**:

| 操作 | 命令 | 说明 |
|------|------|------|
| 索引 | `/memory-hub index` | 显示所有记忆统计 |
| 搜索 | `/memory-hub search <query>` | 跨系统搜索 |
| 存储 | `/memory-hub store <content>` | 保存新记忆 |
| 统计 | `/memory-hub stats` | 显示系统健康状态 |

### 3.4 knowledge-radar Hook

**类型**: Stop Hook

**功能**:
- 会话结束时自动分析高价值信号
- 检测是否需要存储新记忆
- 防止有价值信息丢失

**实现**:
```javascript
// knowledge-radar.js
// 自动分析会话内容，提取高价值记忆
```

**已知问题**:
- Stop Hook 在会话结束时可能重新注入已清理的数据
- 需要定期检查 `settings.json` 中的 hook 配置

### 4.2 数据清理经验

**2026-05-02 数据污染事件处理**:

| 阶段 | 操作 | 结果 |
|------|------|------|
| **发现** | 检测到 192 条记录中包含噪声 | 确认污染 |
| **分析** | 识别 5 类噪声：XML泄漏、转义字符、会话片段、重复内容、边界记录 | 分类完成 |
| **清理** | 批量删除 36 条噪声记录 | 减少至 156 条 |
| **去重** | 删除 20 条重复记录 | 减少 10 组重复 |
| **验证** | 检查清理后状态 | 数据库一致性确认 |

**噪声分类**:

| 类型 | 数量 | 处理方式 |
|------|------|----------|
| XML 标签泄漏 | 17 条 | 批量删除 |
| 转义换行符 | 34 条 | 清理后保留或删除 |
| 会话片段 | 4 条 | 删除 |
| 重复记录 | 10 组 (20 条) | 去重 |
| 边界记录 | 16 条 | 逐条审查决策 |

### 4.3 memU API 问题排查

**发现的问题**:
1. **DELETE 操作未生效**: 删除后记录数异常波动
2. **API 不一致**: HTTP 和 MCP 接口返回数据不一致
3. **检索失败**: `/retrieve` 端点返回空结果但实际有数据

**解决方案**:
- 直接访问 PostgreSQL 数据库进行清理
- 禁用 knowledge-radar Stop Hook 防止数据重新注入
- 修改 `settings.json` 配置 hook 行为
