# UI/UX 相关 Entities 汇总

## 4 个 Design Agents

| Agent | 颜色 | Emoji | 核心职责 |
|-------|------|-------|----------|
| **UX Researcher** | Green | 🔬 | 用户研究、行为分析、可用性测试 |
| **UI Designer** | Purple | 🎨 | 视觉设计系统、组件库、像素级界面 |
| **UX Architect** | Blue | 🏗️ | 技术架构、CSS 系统、实现指导 |
| **Brand Guardian** | #E91E63 | 💎 | 品牌战略、身份一致性、定位 |

## 区别分析

| Agent | 关注点 | 输出 | 阶段 |
|-------|--------|------|------|
| **UX Researcher** | 用户需求与行为 | 研究报告、Persona、测试结果 | 前期调研 |
| **UI Designer** | 视觉呈现与组件 | 设计系统、组件规范 | 设计执行 |
| **UX Architect** | 技术实现与架构 | CSS 架构、代码规范 | 开发准备 |
| **Brand Guardian** | 品牌调性与战略 | 品牌指南、定位策略 | 全局指导 |

## 协作流程

```
Brand Guardian (品牌战略)
        ↓
UX Researcher (用户调研)
        ↓
UI Designer (视觉设计)
        ↓
UX Architect (技术实现)
```

## 关键Insight

1. **设计阶段分工**: 品牌 → 用户 → 视觉 → 技术，环环相扣
2. **上下关系**: Brand Guardian 最抽象（战略），UX Architect 最具体（代码）
3. **循环反馈**: UX Researcher 的发现会影响 UI Designer，UI Designer 的输出是 UX Architect 的输入
