# Receiving-Code-Review 技能使用完全指南

> 来源：obra/superpowers 插件 v5.0.7
> 整理：2026-05-05

## 概述

代码审查需要技术评估，不是情感表演。

**核心原则：** 验证后再实现。询问而不是假设。技术正确性高于社交舒适。

## 响应模式

1. **读取** — 完成反馈而不反应
2. **理解** — 用自己的话重述需求
3. **验证** — 对照代码库实际检查
4. **评估** — 对此代码库技术正确吗？
5. **响应** — 技术确认或合理反驳
6. **实现** — 一次一项，每项测试

## 禁止的响应

**绝不：**
- 你完全正确！
- 好观点！
- 我现在实施

**替代：**
- 重述技术需求
- 询问澄清
- 用技术推理反驳
