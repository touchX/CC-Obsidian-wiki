# Startup MVP Workflow

Comprehensive 4-week workflow for building and launching startup minimum viable products.

## 🎯 Workflow Overview

**Duration**: 4 weeks
**Agents**: 6 specialists
**Goal**: Ship production-ready MVP

## 📅 Weekly Structure

### Week 1: Foundation
| Day | Focus | Agent |
|-----|-------|-------|
| 1-2 | Product strategy | Sprint Prioritizer |
| 3-4 | User research | UX Researcher |
| 5 | Technical architecture | Backend Architect |

### Week 2: Core Build
| Day | Focus | Agent |
|-----|-------|-------|
| 1-2 | Backend development | Backend Architect |
| 3-5 | Frontend development | Frontend Developer |

### Week 3: Polish & Features
| Day | Focus | Agent |
|-----|-------|-------|
| 1-2 | Feature completion | Rapid Prototyper |
| 3-4 | Growth mechanics | Growth Hacker |
| 5 | Integration testing | Reality Checker |

### Week 4: Launch Prep
| Day | Focus | Agent |
|-----|-------|-------|
| 1-2 | User testing | UX Researcher |
| 3-4 | Performance optimization | Frontend Developer |
| 5 | Launch | All Agents |

## 🤖 Agent Team

| Agent | Role | Focus |
|-------|------|-------|
| **Sprint Prioritizer** | Project management | Scope, milestones, priorities |
| **UX Researcher** | User research | Needs, pain points, flows |
| **Backend Architect** | Server-side | APIs, database, infrastructure |
| **Frontend Developer** | Client-side | UI, interactions, responsiveness |
| **Rapid Prototyper** | Iteration | Quick wins, feature validation |
| **Growth Hacker** | Traction | Acquisition, activation, retention |
| **Reality Checker** | Validation | Feasibility, scope, quality |

## 📋 Phase Details

### Week 1: Strategy & Research

**Day 1-2: Sprint Planning**
```
Sprint Prioritizer:
├── Define MVP scope
├── Set weekly milestones
└── Identify blockers early
```

**Day 3-4: User Research**
```
UX Researcher:
├── Interview target users
├── Map user journeys
└── Identify key features
```

**Day 5: Architecture**
```
Backend Architect:
├── Design data model
├── Define API contracts
└── Choose tech stack
```

### Week 2: Core Development

**Focus**: Build the absolute minimum that demonstrates core value

```
Frontend Developer ←→ Backend Architect
     │                      │
     └──────────┬───────────┘
                ▼
          API Integration
```

### Week 3: Polish & Growth

**Focus**: Add growth mechanics and validate with Reality Checker

```
┌─────────────────────────────────────┐
│  Rapid Prototyper → Core Features    │
│  Growth Hacker → Viral Loops         │
│  Reality Checker → Feasibility Audit │
└─────────────────────────────────────┘
```

### Week 4: Launch Readiness

**Focus**: Bug fixes, performance, and go-live

## 📊 Milestone Tracking

| Week | Milestone | Definition of Done |
|------|----------|-------------------|
| 1 | Foundation | Architecture approved, research complete |
| 2 | Core MVP | Core value prop functional |
| 3 | Feature Complete | All planned features built |
| 4 | Launch Ready | Tested, optimized, deployed |

## 💡 Key Principles

- **Week 1 is for learning** — Don't skip research
- **Week 2 is for building** — Ruthless focus on core value
- **Week 3 is for validating** — Reality check every assumption
- **Week 4 is for shipping** — Lock scope, polish and deploy
