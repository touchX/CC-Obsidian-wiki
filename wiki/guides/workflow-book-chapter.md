# Book Chapter Workflow

Single-agent workflow for transforming rough notes and素材 into polished first-person chapter drafts.

## 🎯 Workflow Overview

**Purpose**: Convert unorganized thoughts into compelling book chapters
**Agent**: Book Co-Author specialist
**Output**: Publication-ready chapter draft

## 📋 Execution Flow

```
Rough Notes/素材
       ↓
┌──────────────────┐
│  Book Co-Author  │
│  ──────────────  │
│  1. Analyze voice│
│  2. Structure arc│
│  3. Draft chapter│
│  4. Polish prose │
└──────────────────┘
       ↓
Publication-Ready Draft
```

## 🔄 Detailed Steps

### Step 1: Voice Analysis

Identify the author's:
- Writing style and tone
- Signature phrases and metaphors
- Target audience
- Perspective and expertise

### Step 2: Arc Construction

Build chapter structure:
- Opening hook
- Core argument or story
- Supporting evidence
- Memorable close

### Step 3: Draft Generation

Write first-person content:
- Authentic voice throughout
- Narrative flow
- Appropriate pacing
- Engagement hooks

### Step 4: Polish

Refine for publication:
- Prose clarity
- Consistency check
- Formatting for publishing
- Final read-through

## 💡 Key Techniques

| Technique | Purpose |
|-----------|---------|
| Voice cloning | Match author's tone |
| Story injection | Make concepts relatable |
| Evidence weaving | Support with examples |
| Cliffhangers | Maintain reader interest |

## 📝 Input Example

```
Raw notes:
"users don't understand AI, need explainers, 
trust issues, transparency matters, 
show working not just results"
```

**Transforms to**:
> "When users see an AI recommendation, they often ask: 'Why should I trust this?' It's a fair question—and honestly, one the AI industry has done a poor job answering. Let me show you a different approach..."
