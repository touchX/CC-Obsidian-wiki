## 核心问题与解决方案

### 两个核心问题

| 问题 | 解答 |
|------|------|
| Harness 有什么具体的工程实践？ | 将视频生成流程封装成 Skill |
| 视频效果到底是怎么做的？ | AI 生成网页（非视频生成模型） |

### 为什么用网页而不是视频模型

| 方案 | 问题 | 结论 |
|------|------|------|
| **视频生成模型** | 抽卡不稳定，成本高 | ❌ |
| **Notebook LM** | 只能生成静态图，无法动画演示 | ❌ |
| **Remotion 框架** | 限制模型发挥，不够自由 | ❌ |
| **Web Coding（自选）** | 完全可控，灵活定制 | ✅ |

> [!tip] 核心理念
> **可控性 > 自动化**：字体、配色、停留时间、精确数字显示，网页里改几行代码就能搞定。

## 视频效果实现要点

### 四个关键步骤

| 步骤 | 说明 | 关键点 |
|------|------|--------|
| **1. 文章转口播稿** | 书面化 → 口语化 | 短句、人称化、聊天感 |
| **2. 拆解开发大纲** | 口播稿 → 具体步骤 | 每段话对应一个画面 |
| **3. 视觉演示** | 每一步生成动态效果 | 抽象概念可视化 |
| **4. 节奏同步** | 网页跟随口播稿节奏 | 自然流畅 |

### 网页设计特点

```
┌─────────────────────────────────────────┐
│              干净画面                    │
│   （无页眉、无页码、无品牌标识）          │
├─────────────────────────────────────────┤
│                                         │
│          动态演示效果                   │
│     （跟随口播稿节奏自动播放）            │
│                                         │
├─────────────────────────────────────────┤
│              进度条                     │
│       （鼠标浮现时显示）                  │
├─────────────────────────────────────────┤
│     章节跳转 | 步骤切换 | 进度控制        │
└─────────────────────────────────────────┘
```

## Harness 驾驭工程

### 核心问题

> 模型有能力，怎么让它每次都做得出来？

### 模型困境

| 问题 | 表现 |
|------|------|
| **稿画不对** | 口播稿和画面不匹配 |
| **章节不一致** | 章节数和音频数不匹配 |
| **风格突变** | 后面章节风格与前面完全不同 |

### Harness 解决方案

Harness 是一套机制，用来驾驭 Agent 的能力：

| 机制 | 作用 |
|------|------|
| **划定边界** | 明确工作范围 |
| **设定检查点** | 阶段性验证 |
| **自检和修复** | 自动纠正偏差 |

> [!tip] Harness 本质
> Harness 是一份给 Agent 的操作手册，告诉它什么时候该做什么、做到什么标准、哪些红线不能碰。

## Web Video Presentation Skill

### Skill 概述

将文章 → 口播稿 → 开发大纲 → 视觉演示 → 网页视频的完整流程封装成可复用的 Skill。

### 架构设计

```
输入：技术文章
    ↓
Skill: Web Video Presentation
    ↓
┌─────────────────────────────────────┐
│ 1. 转化为口播稿                     │
│ 2. 拆解开发大纲（章节+步骤）         │
│ 3. 生成视觉演示代码                  │
│ 4. 同步节奏控制                     │
│ 5. 自检和修复                       │
└─────────────────────────────────────┘
    ↓
输出：精美网页视频
```

### 核心特性

| 特性 | 说明 |
|------|------|
| **章节拆分** | 将文章拆成 N 个章节 |
| **步骤细化** | 每个章节包含多个细粒度步骤 |
| **进度控制** | 底部进度条，鼠标浮现显示 |
| **任意跳转** | 可跳转到任意章节和步骤 |
