# RAG 工作机制详解

基于马克的技术工作坊视频教程，BVID: BV1JLN2z4EZQ，时长 17 分钟。

## 课程概述

本教程系统讲解 RAG（Retrieval Augmented Generation，检索增强生成）的核心概念、实现原理和五大核心阶段。

**学习目标**：
1. 理解 RAG 的定义和使用场景
2. 掌握 RAG 的五大核心阶段：分片、索引、召回、重排、生成
3. 学习向量、Embedding、向量数据库等核心概念
4. 理解向量相似度计算方法（余弦相似度、欧式距离、点积）

## RAG 概念定义

### 什么是 RAG

> [!tip] 核心定义
> **RAG** = **R**etrieval **A**ugmented **G**eneration（检索增强生成）
>
> 核心思想：先从资料库里检索相关的内容，再基于这些内容来生成答案。

**类比理解**：
- **传统问答** = 直接给模型整本手册 → 成本高、速度慢、准确率低
- **RAG 问答** = 只检索相关片段给模型 → 成本低、速度快、准确率高

**现实世界案例**：
- 企业知识助手
- 智能客服系统
- 产品文档问答
- 技术支持机器人

### RAG 解决的问题

| 问题 | 传统方案问题 | RAG 解决方案 |
|------|-------------|-------------|
| **上下文窗口限制** | 模型无法读取过长文档 | 只发送相关片段 |
| **推理成本高** | 每次都发送完整文档 | 只发送少量片段 |
| **推理速度慢** | 需要处理大量内容 | 处理内容大幅减少 |

## RAG 五大核心阶段

### 整体架构

```
┌─────────────────────────────────────────────────┐

│                                                  │

│  ┌──────────────────────────────────────┐      │
│  │ 1. 分片 (Chunking)                   │      │
│  │    按字数/段落/章节切分文档          │      │
│  └──────────────────────────────────────┘      │
│    ↓                                             │
│  ┌──────────────────────────────────────┐      │
│  │ 2. 索引 (Indexing)                   │      │
│  │    Embedding → 向量 → 向量数据库      │      │
│  └──────────────────────────────────────┘      │
│                                                  │
│  【提问后 - 问答流程】                           │
│    ↓                                             │
│  ┌──────────────────────────────────────┐      │
│  │ 3. 召回 (Retrieval)                  │      │
│  │    向量相似度 → Top 10 片段          │      │
│  └──────────────────────────────────────┘      │
│    ↓                                             │
│  ┌──────────────────────────────────────┐      │
│  │ 4. 重排 (Reranking)                  │      │
│  │    CrossEncoder → Top 3 片段         │      │
│  └──────────────────────────────────────┘      │
│    ↓                                             │
│  ┌──────────────────────────────────────┐      │
│  │ 5. 生成 (Generation)                 │      │
│  │    LLM 基于片段生成答案              │      │
│  └──────────────────────────────────────┘      │
│                                                  │
└─────────────────────────────────────────────────┘
```

## 第一阶段：分片（Chunking）

### 基本概念

分片是将文档切分成多个片段的过程。

### 分片方式

| 方式 | 说明 | 示例 |
|------|------|------|
| **按字数** | 固定字符数切分 | 1000 字/片段 |
| **按段落** | 自然段落切分 | 1 段落/片段 |
| **按章节** | 标题层级切分 | 1 章节/片段 |
| **按页码** | PDF 页面切分 | 1 页/片段 |

### 分片原则

> [!tip] 最佳实践
> - 片段不宜过长（影响检索精度）
> - 片段不宜过短（缺乏上下文）
> - 保持语义完整性（避免截断句子）
> - 重叠切分（保留边界上下文）

## 第二阶段：索引（Indexing）

### 核心概念

索引是通过 Embedding 将片段文本转换为向量，并存储到向量数据库的过程。

### 三个关键概念

#### 1. 向量（Vector）

**数学定义**：有大小有方向的量，用数组表示

**维度概念**：
- 一维向量：`[1]`
- 二维向量：`[1, 2]`
- 三维向量：`[1, 2, 3]`
- 高维向量：`[v1, v2, ..., vn]`（几百到几千维）

**RAG 中的向量**：
- 通常为几百到几千维
- 维度越大，包含信息越丰富
- 无法直接可视化，但可计算

#### 2. Embedding

**定义**：将文本转换为向量的过程

**核心特性**：语义相近的文本 → 向量也相近

**示例**：
```
"马克喜欢吃水果"  → [1, 2]
"马克爱吃水果"    → [1, 1]     （相近）
"天气真好"       → [3, -1]    （远离）
```

**Embedding 模型**：
- 专门的模型（非 GPT-4o/Claude 等）
- 参考 MTEP 排行榜选择模型
- 成本低、速度快

#### 3. 向量数据库（Vector Database）

**定义**：专门存储和查询向量的数据库

**核心功能**：
- 存储向量 + 原始文本
- 计算向量相似度
- 高效检索

**表结构示例**：
```
┌─────────────────┬──────────────┐
│ 原始文本        │ 向量         │
├─────────────────┼──────────────┤
│ "马克喜欢吃水果"│ [1, 2]       │
│ "天气真好"      │ [3, -1]      │
└─────────────────┴──────────────┘
```

### 索引流程

```
片段1 → Embedding模型 → 向量1 → 向量数据库
片段2 → Embedding模型 → 向量2 → 向量数据库
片段3 → Embedding模型 → 向量3 → 向量数据库
...
```

> [!important] 关键点
> 向量只是中间结果，最终需要的还是原始文本。

## 第三阶段：召回（Retrieval）

### 基本概念

召回是搜索与用户问题相关片段的过程。

### 召回流程

```
用户问题
    ↓
Embedding 模型
    ↓
问题向量
    ↓
向量数据库（计算相似度）
    ↓
Top 10 相关片段
```

### 向量相似度计算

#### 计算方法

| 方法 | 原理 | 特点 |
|------|------|------|
| **余弦相似度** | 计算两个向量夹角的 cos 值 | 夹角越小，相似度越高 |
| **欧式距离** | 计算两个向量的距离 | 距离越小，相似度越高 |
| **点积（Dot Product）** | 向量代数乘积 | 方向一致且越长，值越大 |

#### 计算示例

```
用户问题向量：[u1, u2, ...]
片段1向量：  [c1, c2, ...]
片段2向量：  [c3, c4, ...]
片段3向量：  [c5, c6, ...]

相似度(问题, 片段1) = 计算公式([u1, u2], [c1, c2])
相似度(问题, 片段2) = 计算公式([u1, u2], [c3, c4])
相似度(问题, 片段3) = 计算公式([u1, u2], [c5, c6])

排序 → 取 Top 10
```

### 召回特点

| 特性 | 说明 |
|------|------|
| **成本** | 低 |
| **速度** | 快 |
| **准确率** | 较低 |
| **适用场景** | 初步筛选，从大量片段中筛选 |

## 第四阶段：重排（Reranking）

### 基本概念

重排是从召回的 Top 10 片段中，再精选 Top 3 片段的过程。

### 为什么需要重排？

> [!tip] 类比理解
> **公司招聘流程**：
> - **召回** = 简历筛选（粗略方法，从成千上万份中选 10 人）
> - **重排** = 面试（仔细评估，从 10 人中选 3 人）

### 重排模型：CrossEncoder

| 特性 | 召回（向量相似度） | 重排（CrossEncoder） |
|------|------------------|---------------------|
| **成本** | 低 | 高 |
| **速度** | 快 | 慢 |
| **准确率** | 低 | 高 |
| **用途** | 初步筛选 | 精挑细选 |

### 重排流程

```
召回的 Top 10 片段
    ↓
CrossEncoder 模型
    ↓
重新计算相似度
    ↓
Top 3 最相关片段
```

### 重排优势

- 准确率大幅提升
- 最终结果质量更高
- 虽然耗时，但只处理少量片段

## 第五阶段：生成（Generation）

### 基本流程

```
用户问题 + Top 3 相关片段
    ↓
大语言模型（LLM）
    ↓
生成最终答案
```

### 生成特点

- LLM 基于相关片段生成答案
- 只处理少量高质量内容
- 答案准确率高、相关性强

## RAG 完整流程示例

### 提问前（数据准备）

```
产品手册（100页）
    ↓
分片 → 500 个片段
    ↓
Embedding → 500 个向量
    ↓
向量数据库（存储向量+原始文本）
```

### 提问后（问答流程）

**用户问题**："这个产品支持哪些支付方式？"

```
1. Embedding → 问题向量
2. 向量数据库 → Top 10 片段（向量相似度）
3. CrossEncoder → Top 3 片段（精确排序）
4. LLM → "该产品支持信用卡、支付宝、微信支付"
```

## 技术栈总结

### 核心组件

| 组件 | 作用 | 常用工具/模型 |
|------|------|--------------|
| **Embedding 模型** | 文本转向量 | 参考 MTEP 排行榜 |
| **向量数据库** | 存储和检索向量 | Pinecone, Milvus, Weaviate |
| **相似度计算** | 计算向量相似度 | 余弦/欧式/点积 |
| **CrossEncoder** | 重排序 | BGE-Reranker, Cohere Rerank |
| **LLM** | 生成答案 | GPT-4o, Claude, DeepSeek |

### 性能优化

| 阶段 | 优化策略 |
|------|----------|
| **分片** | 合理设置片段大小、保留上下文 |
| **索引** | 选择高质量 Embedding 模型 |
| **召回** | 选择合适的相似度计算方法 |
| **重排** | 调整 Top K 值（10→3） |
| **生成** | 精简 Prompt、选择合适 LLM |
