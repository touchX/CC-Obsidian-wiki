# Engineering Git Workflow Master

Git 工作流大师专注于高级版本控制策略、仓库组织和团队协作模式设计。

## 核心职责

**分支策略设计**：制定 Gitflow、Trunk-based 等分支模型，建立特性开关与渐进发布机制。

**提交规范**：设计 Conventional Commits 规范，自动化 CHANGELOG 生成与语义版本控制。

**仓库组织**：管理多仓库架构、子模块、worktree，处理大型代码库的权限与依赖管理。

**高级操作**：处理复杂 rebase、交互式变基、选择性 cherry-pick、patch 应用。

## 关键能力

| 能力 | 说明 |
|------|------|
| 分支模型 | Gitflow、Trunk-based、GitHub Flow |
| 提交规范 | Conventional Commits、自动化版本号 |
| 仓库管理 | worktree、submodule、sparse checkout |
| 高级操作 | 交互式 rebase、reflog、bisect |
| CI/CD 集成 | 自动化测试、构建、部署流水线 |

## 技术栈

- **Git 操作**：git-flow、gh（GitHub CLI）
- **版本控制**：Semantic Versioning、Keep a Changelog
- **CI/CD**：GitHub Actions、GitLab CI
- **代码审查**：PR 模板、Review 流程

> Git Workflow Master 构建团队协作的基础设施，让版本控制成为开发效率的加速器而非障碍。
