# Vercel v0 编程助手

## 概述

v0 是 Vercel 开发的 AI 编程助手，专注于 Web 应用程序开发。它基于 Next.js 和 AI SDK，为前端开发者提供从组件生成到完整页面创建的端到端服务。v0 的核心理念是通过 AI 驱动的设计系统，让开发者能够快速构建高质量的 Web 界面。

## 核心定位

| 维度 | 说明 |
|------|------|
| **开发者** | Vercel |
| **运行环境** | v0.dev |
| **底层框架** | Next.js |
| **AI SDK** | AI SDK by Vercel |
| **核心能力** | 组件生成、页面创建、设计系统 |

## 设计系统

### 核心原则

v0 遵循统一的 Web 设计标准：

| 原则 | 说明 |
|------|------|
| **UI 框架** | shadcn/ui |
| **颜色数量** | 3-5 个核心颜色 |
| **字体数量** | 最多 2 种字体 |
| **设计风格** | 移动优先、响应式 |

### shadcn/ui 规范

shadcn/ui 是 v0 的默认 UI 框架：

```tsx
import { Button } from "@/components/ui/button"
import { Card } from "@/components/ui/card"

// 基于 Radix UI 的无头组件
// 完全可定制样式
```

**组件特点**：
- 不是组件库，是可复制粘贴的代码
- 基于 Radix UI 原语
- 使用 Tailwind CSS 样式
- 暗色模式支持

### 颜色设计

```typescript
// 建议的配色方案
colors: {
  primary: "#3b82f6",   // 蓝色
  secondary: "#64748b",  // 灰蓝色
  accent: "#f59e0b",    // 琥珀色
  background: "#ffffff", // 白色
  foreground: "#0f172a", // 深蓝黑

```

### 字体搭配

| 用途 | 推荐字体 |
|------|----------|
| 标题 | Inter / Geist Sans |
| 正文 | system-ui / 平台默认 |

## Next.js 16 新特性

v0 充分利用 Next.js 最新特性：

### 代理路由 (proxy.js)

```javascript
// proxy.js
export async function GET(request) {
  const url = new URL(request.url)
  const targetUrl = url.searchParams.get('url')
  
  const response = await fetch(targetUrl)
  return new Response(response.body, {
    headers: response.headers,
  })
}
```

### Turbopack

新一代打包工具：
- Rust 实现
- 10x 更快启动
- 增量编译

### React Compiler

自动优化 React 代码：
- 无需手动 useMemo/useCallback
- 自动转换 hooks
- 保留手动优化能力

### 缓存指令

```tsx
// 服务端缓存
async function getData() {
  "use server"
  return await fetchData()
}

// 客户端缓存刷新
import { useRevalidate } from 'ai/react'

function RefreshButton() {
  const revalidate = useRevalidate()
  return <button onClick={() => revalidate('/api/data')}>
    刷新数据
  </button>
}
```

## AI SDK 缓存 API

Vercel AI SDK 提供强大的缓存能力：

### revalidateTag

```typescript

```

### updateTag

```typescript

// 更新特定标签的数据

  name: 'John',
  email: 'john@example.com'

```

### refresh

```typescript

// 刷新当前页面
refresh()
```

## 工具集详解

v0 提供 5 个核心工具。

### 1. AskUserQuestions

向用户提出澄清性问题。

```typescript
{
  "questions": [
    {
      "text": "您想要哪种颜色主题？",
      "options": ["浅色", "深色", "系统跟随"],
      "header": "主题"
    }
  ]
}
```

**触发场景**：
- 需求不明确时
- 需要设计决策时
- 技术选型确认时

### 2. Move

移动文件到新位置。

```typescript
{
  "source": "components/Button.tsx",
  "destination": "ui/Button.tsx"
}
```

**效果**：
- 自动更新所有导入引用
- 保持 git 历史

### 3. GenerateImage

使用 AI 生成图片。

```typescript
{
  "prompt": "A modern dashboard UI with dark theme",
  "aspectRatio": "16:9",
  "style": "minimal"
}
```

**用途**：
- 设计稿预览
- 图标和插图
- 背景图片

### 4. Write

写入或覆盖文件。

```typescript
{
  "path": "src/app/page.tsx",
  "content": "// 文件内容..."
}
```

**规则**：
- 完整覆盖文件
- 自动创建目录

### 5. scripts

执行脚本命令。

```typescript
{
  "script": "python prepare_data.py",
  "timeout": 60000
}
```

**支持**：
- Python 脚本
- Node.js 脚本
- Shell 命令

## 组件开发流程

### 1. 接收需求

```
用户: 创建一个用户资料卡片组件
```

### 2. 澄清问题

```typescript
AskUserQuestions: {
  questions: [{
    text: "卡片需要展示哪些信息？",
    options: ["仅头像和名字", "完整资料", "带社交链接"]
  }]
}
```

### 3. 生成代码

```typescript
Write: {
  path: "components/UserCard.tsx",
  content: `export function UserCard({ user }) {
    return (
      <Card className="p-4">
        <Avatar src={user.avatar} />
        <h3>{user.name}</h3>
        <p>{user.bio}</p>
      </Card>
    )
  }`
}
```

### 4. 添加样式

使用 Tailwind CSS 和 shadcn/ui：

```tsx
<Card className="w-[300px] p-4 hover:shadow-lg transition-shadow">
  <Avatar className="w-16 h-16" src={user.avatar} />
  <h3 className="text-lg font-semibold">{user.name}</h3>
  <p className="text-muted-foreground">{user.bio}</p>
</Card>
```

## 工作流示例

### 创建登录页面

```
用户: 创建一个现代化的登录页面

v0:
1. AskUserQuestions: 确认设计风格和功能需求
2. Write: 创建 app/login/page.tsx
3. Write: 创建相关组件
4. Write: 添加样式
5. 返回预览
```

### 添加暗色模式

```
用户: 为现有页面添加暗色模式支持

v0:
1. Write: 更新 tailwind.config.js
2. Write: 添加 ThemeProvider
3. Write: 更新组件样式变量
4. 返回更新后的页面
```
