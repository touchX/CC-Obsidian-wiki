# Manus AI 智能代理

## 概述

Manus 是由 Manus 团队创建的先进 AI 代理系统，代表了通用型 AI 代理的重要突破。它能够在 sandbox 环境中完全自主地处理复杂任务，从数据分析到代码开发，从信息检索到内容创作。Manus 的核心理念是将 AI 能力封装为可执行的智能工具，让机器能够像人类一样思考、规划和执行多步骤任务。

## 核心定位

| 维度 | 说明 |
|------|------|
| **开发者** | Manus 团队 |
| **架构** | 事件驱动型智能代理 |
| **运行环境** | Sandbox（Ubuntu 22.04） |
| **编程语言** | Python 3.10.12 / Node.js 20.18.0 |
| **调用格式** | OpenAI function calling |

## 智能代理循环

Manus 采用六阶段迭代执行模型，每个阶段都有明确的目标和产出物。代理循环确保任务能够被系统性地分解、执行和验证。

```
┌─────────────────────────────────────────────────────────────────┐
│                     Manus Agent Loop                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   阶段 1: 分析事件 ────▶ 阶段 2: 选择工具                         │
│         │                         │                              │
│         │                         ▼                              │
│         │                   阶段 3: 等待执行                      │
│         │                         │                              │
│         ▼                         ▼                              │
│   阶段 6: 进入待机 ◀── 阶段 4: 迭代 ◀── 阶段 5: 提交结果          │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 阶段详细说明

**阶段 1 - 分析事件**：系统接收来自事件流的信息，包括用户消息、文件操作结果、命令执行输出等。分析器解析这些事件的类型、优先级和关联性，建立当前执行上下文。分析结果将决定下一步应采取的行动。

**阶段 2 - 选择工具**：基于分析结果，从 26 个可用工具中选择最适合当前任务的工具。工具选择考虑任务类型、时间效率、准确性要求等因素。选中的工具将被调用并传入相应的参数。

**阶段 3 - 等待执行**：工具调用被发送到 sandbox 环境执行。在此期间，系统监控执行状态，处理可能的错误和超时情况。执行完成后，结果将被收集并转换为标准格式。

**阶段 4 - 迭代**：对于复杂任务，单次工具调用可能不足以完成任务。此时系统评估当前状态，决定是否需要再次调用工具或进入下一阶段。迭代过程持续到任务完成或达到最大迭代次数。

**阶段 5 - 提交结果**：所有必要的工具调用和迭代完成后，系统整理最终结果。结果以结构化格式呈现，包含执行摘要、关键发现和交付物。

**阶段 6 - 进入待机**：任务完成后，代理进入待机状态等待下一条消息。待机期间保持上下文活跃，可以快速响应后续请求。

## 事件流系统

事件流是 Manus 与外部环境通信的媒介，包含六种事件类型。

```
事件类型 ──────────────── 说明
     │
     ├─ Message      用户消息和代理响应
     ├─ Action       工具调用请求
     ├─ Observation  执行结果和状态更新
     ├─ Plan         规划和决策信息
     ├─ Knowledge    知识库查询结果
     ├─ Datasource   数据源连接状态
     └─ Other        其他辅助信息
```

### 事件处理流程

当事件流中产生新事件时，分析器首先判断事件类型。对于 Message 类型事件，系统提取任务描述和约束条件。对于 Action 类型事件，系统记录工具调用意图。对于 Observation 类型事件，系统更新内部状态并触发后续处理。这种分类处理确保了系统能够正确响应各种输入。

## 系统模块架构

Manus 的系统提示包含十二个核心模块，每个模块负责特定功能。

### 规划器模块

规划器负责将复杂任务分解为可执行的子任务。它维护任务队列，支持任务依赖关系的定义，并跟踪任务执行进度。规划器输出结构化的任务计划，包含优先级、完成标准和时间估计。

规划器的工作基于任务分析和策略选择。当接收到新任务时，规划器首先评估任务复杂度，然后决定采用线性计划还是迭代计划。线性计划适用于简单明确的单路径任务，迭代计划适用于需要探索和适应的复杂任务。

规划器的输出通常包含任务列表、依赖关系、执行顺序和预期结果。这些信息被传递给事件处理器进行下一步处理。

### 知识模块

知识模块管理代理的知识库访问。它提供语义搜索能力，支持从外部知识源检索相关信息。知识模块维护索引结构，实现快速检索和相关性排序。

当执行任务需要特定领域知识时，知识模块被调用。它接受查询请求，搜索匹配的知识条目，返回格式化结果。知识模块还支持增量学习，可以将新的信息添加到知识库中。

知识模块与规划器紧密协作。在任务分解阶段，规划器可能查询知识模块获取相关背景信息。在执行阶段，知识模块提供领域专业知识支持。

### 数据源模块

数据源模块管理外部数据连接。它支持 API 调用、数据库查询、文件读取等多种数据访问方式。数据源模块维护连接池，处理认证和错误恢复。

```python
# ApiClient 使用示例
from datasource import ApiClient

client = ApiClient(api_key="your-api-key")
response = client.get(endpoint="/api/data", params={"limit": 100})
data = response.json()
```

数据源模块的设计强调可靠性和灵活性。它实现了自动重试机制，对于临时性连接问题会自动重试。对于持续性错误，模块会记录错误详情并返回有意义的错误消息。

### 待办事项规则

待办事项（todo.md）是 Manus 内部任务跟踪的核心机制。每个任务被表示为一个待办项，包含描述、状态、优先级和截止日期等信息。

```
todo.md 结构：
[ ] 任务描述 - 状态: pending, 优先级: high
[x] 已完成任务 - 状态: completed
[>] 进行中任务 - 状态: in_progress
[!] 阻塞任务 - 状态: blocked
```

待办事项规则定义了如何创建、更新和完成待办项。规划器在任务分解时创建待办项，执行器在完成子任务后更新状态。这种显式跟踪机制确保了复杂任务的可靠执行。

### 消息规则

消息规则定义了代理与用户通信的格式和时机。代理在关键节点发送消息，包括任务开始、执行进度、重要发现和最终结果。消息内容保持专业、简洁和可操作性。

消息规则还定义了如何处理用户反馈。当收到用户修正或补充时，系统评估其影响并相应调整执行计划。重要反馈被记录到日志中用于后续分析。

### 文件规则

文件规则管理 sandbox 环境中的文件操作。所有文件操作通过标准化接口进行，支持创建、读取、修改和删除操作。文件规则还定义了路径规范和权限控制。

文件操作的执行遵循原子性原则。关键文件操作前，系统创建备份以便回滚。文件规则确保所有文件操作可追溯和可审计。

### 浏览器规则

浏览器规则定义了 Web 交互的规范。代理可以导航网页、填写表单、点击元素和提取内容。浏览器规则强调安全性和效率，避免无意义的页面加载和重复操作。

浏览器操作使用无头模式以提高性能。系统维护浏览器状态缓存，减少页面重新加载。对于复杂交互，代理先分析页面结构，然后执行精确的元素定位和操作。

## 工具集详解

Manus 提供 26 个工具，采用 OpenAI function calling 格式定义。所有工具按功能分为七大类别。

| 类别 | 工具数量 | 主要功能 |
|------|----------|----------|
| 消息工具 | 2 | 用户通信、状态更新 |
| 文件工具 | 5 | 文件创建、读取、搜索、修改 |
| Shell 工具 | 5 | 命令执行、进程管理 |
| 浏览器工具 | 11 | Web 导航、表单填写、内容提取 |
| 信息工具 | 1 | 知识查询 |
| 部署工具 | 2 | 应用部署、Web 发布 |
| 其他工具 | 2 | 待机、页面制作 |

### 消息类工具

消息类工具用于代理与用户的通信。

**send_message_to_user**：向用户发送格式化消息，支持文本和结构化数据。消息可以是简单文本回复，也可以是包含多个字段的复杂响应。工具自动处理消息的格式化和发送状态跟踪。

**update_task_status**：更新任务执行状态。状态更新包括进度百分比、当前阶段、预计完成时间和遇到的障碍。这些信息帮助用户了解任务执行情况。

### 文件类工具

文件类工具支持 sandbox 环境中的文件操作。

**read_file**：读取指定文件的内容。工具支持行范围指定，可以读取整个文件或特定行段。读取结果被转换为标准格式，便于后续处理。

**read_multiple_files**：批量读取多个文件。当需要处理多个相关文件时，批量读取比逐个读取更高效。工具确保所有文件成功读取或返回完整的错误报告。

**append_to_file**：向现有文件追加内容。对于日志文件和分析报告，追加写入比完全重写更安全。工具处理文件不存在的情况，可以选择创建新文件或返回错误。

**write_to_file**：创建新文件或覆盖现有文件。对于新文件创建，工具确保父目录存在。对于覆盖操作，工具先创建临时备份以便恢复。

**search_files**：根据模式搜索文件。搜索支持文件名匹配和内容搜索两种模式。结果按相关度排序，并显示文件路径和匹配摘要。

### Shell 类工具

Shell 类工具在 sandbox 环境中执行命令。

**execute_command**：在 sandbox 中执行 Shell 命令。工具支持命令超时设置、环境变量配置和工作目录指定。执行结果包含标准输出、标准错误和退出码。

**execute_command_with_output**：带输出重定向的命令执行。工具将命令输出重定向到指定文件，便于处理大量输出或需要持久化的日志。

**get_current_date**：获取当前日期。工具返回格式化日期字符串，支持多种格式配置。日期信息对于时间敏感的任务至关重要。

**get_current_time**：获取当前时间戳。工具返回精确到秒的时间戳，支持时区配置。时间戳用于日志分析和执行性能测量。

**get_current_datetime**：获取完整日期时间信息。工具返回包含日期、时间、时区的完整信息。方便日志记录和跨时区任务处理。

### 浏览器类工具

浏览器类工具提供 Web 交互能力。

**browse_to_url**：导航到指定 URL。工具处理页面加载超时和错误重试。加载完成后，工具返回页面内容和元数据。

**get_browser_elements**：获取页面元素列表。工具分析 DOM 结构，提取可交互元素。每个元素包含类型、标识符、位置和可见性信息。

**click_element**：点击页面元素。工具支持多种定位方式，包括 ID、类名、CSS 选择器和相对位置。点击前验证元素可点击状态。

**fill_form_input**：填写表单字段。工具处理输入验证和特殊字符转义。支持文本、密码、日期等多种输入类型。

**hover_over_element**：鼠标悬停到元素上。悬停可以触发下拉菜单、工具提示等动态效果。工具记录悬停期间的页面变化。

**get_element_text**：获取元素文本内容。工具提取元素的显示文本，支持嵌套元素的文本合并。返回纯文本，不包含 HTML 标签。

**get_element_attribute**：获取元素属性值。支持所有标准 HTML 属性，包括 value、href、src 等动态属性。属性值用于进一步的操作定位。

**scroll_element_into_view**：滚动元素到可视区域。对于长页面，工具确保目标元素可见再进行操作。滚动动画平滑，用户体验友好。

**wait_for_element**：等待元素出现或消失。工具支持超时设置和条件组合。等待可以基于元素存在性、可见性或属性变化。

**go_back_in_browser**：返回浏览器历史上一页。工具处理导航边界情况。当无历史记录时，返回有意义的错误消息。

**refresh_current_page**：刷新当前页面。工具清除缓存重新加载，适用于内容更新的场景。

### 信息类工具

信息类工具提供知识查询能力。

**search_knowledge_base**：搜索知识库获取相关信息。搜索支持语义匹配和关键词过滤。结果按相关性排序，显示摘要和来源。

### 部署类工具

部署类工具支持应用发布。

**deploy_static_website**：部署静态网站到目标环境。工具处理资源压缩、传输和配置更新。部署完成后返回访问地址和状态。

**make_manus_page**：创建 Manus 格式的 MDX 页面。这是一个特殊工具，用于生成 Manus 平台兼容的内容页面。

### 其他工具

**idle**：进入待机状态。工具暂停代理活动，释放系统资源。待机状态可以被新消息唤醒。

**get_workspace_dir**：获取当前工作目录路径。工具返回 sandbox 环境的工作目录绝对路径。

## Sandbox 环境

Manus 在隔离的 sandbox 环境中执行任务，确保安全性和可控性。

### 环境配置

```
操作系统: Ubuntu 22.04
Python 版本: 3.10.12
Node.js 版本: 20.18.0
工作目录: /workspace
```

Ubuntu 22.04 提供稳定的基础设施和广泛的工具支持。Python 3.10.12 支持现代 Python 语法和丰富的库生态。Node.js 20.18.0 支持 JavaScript/TypeScript 开发需求。

### 环境限制

Sandox 环境对资源使用进行限制，包括 CPU 时间、内存占用和网络访问。这些限制确保多任务环境中的公平性和稳定性。资源限制以配置文件形式管理，可以根据任务需求动态调整。

## 写作规范

Manus 在输出内容时遵循严格的写作规范，确保交付物的专业性和可读性。

### 格式规则

**避免使用列表格式**：相对于项目符号列表，Manus 倾向于使用连贯的段落叙述。这种风格更适合深度内容表达，使观点之间的逻辑关系更加清晰。

```
禁止模式：
- 不要使用连续的项目符号
- 不要使用简短的要点堆砌
- 不要使用表格作为主要呈现方式

推荐模式：
- 使用自然段落展开论述
- 使用分层标题组织结构
- 使用示例和案例说明概念
```

**最小字数要求**：对于重要文档，单个文档的输出应不少于数千字。这种篇幅要求确保内容深度，避免表面化的泛泛而谈。

**段落连贯性**：每个段落应该是完整的内容单元，拥有独立的主题和支撑论据。段落之间通过过渡句连接，形成流畅的阅读体验。

### 内容质量

内容应该体现专业性和洞察力。避免显而易见的陈述，尝试提供独特的视角和深入的分析。当讨论技术概念时，应该解释原理而非仅仅描述表面现象。

## 错误处理

系统实现了多层错误处理机制。

### 工具调用错误

当工具调用失败时，系统首先评估错误类型。瞬时错误（如网络超时）触发自动重试，永久错误（如权限不足）立即报告给用户。对于需要人工干预的错误，系统提供详细的错误信息和可能的解决建议。

### 执行超时

长时间运行的命令被监控执行时间。当达到超时阈值时，系统发送终止信号并记录执行状态。用户可以选择重新尝试或修改任务范围。

### 状态恢复

系统维护执行状态的持久化。当 sandbox 重启或连接中断时，系统从上一个检查点恢复执行。这种机制确保了长时间任务的可靠性。
