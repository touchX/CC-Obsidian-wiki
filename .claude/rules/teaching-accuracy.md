# 教学准确性强制检查清单

> **触发条件**：每次创建教学文档、回答技术问题、或提供代码示例时
> **强制执行**：不允许跳过任何检查项

## 🚨 红灯检查项（违反即停止）

- [ ] **1. Wiki-First 验证**
  - [ ] 是否已查询 `wiki/index.md`？
  - [ ] 是否已读取相关的 Wiki 页面？
  - [ ] 如果没有相关 Wiki 页面，是否明确说明"基于训练数据"？

- [ ] **2. 事实来源验证**
  - [ ] 每个技术事实是否有可引用的来源？
  - [ ] 代码示例是否基于项目中的真实文件？
  - [ ] 如果基于训练数据，是否明确标注"未经验证"？

- [ ] **3. 实际示例验证**
  - [ ] 教学的语法/格式是否与项目中的真实示例一致？
  - [ ] 是否读取了 `.claude/commands/` 或 `.claude/agents/` 中的实际文件？
  - [ ] 如果示例文件不存在，是否明确说明"这是示例，非项目实际文件"？

## 🟡 黄灯检查项（警告但允许继续）

- [ ] **4. 一致性检查**
  - [ ] 新内容是否与 `CLAUDE.md` 中的规则一致？
  - [ ] 是否与已有的 Wiki 页面内容冲突？
  - [ ] 如果有冲突，是否在文档中说明？

- [ ] **5. 完整性检查**
  - [ ] 是否提供了足够的上下文？
  - [ ] 是否说明了适用场景和限制？
  - [ ] 是否提供了相关资源的链接？

## ✅ 绿灯检查项（最佳实践）

- [ ] **6. 质量提升**
  - [ ] 是否引用了具体文件路径和行号？
  - [ ] 是否提供了可运行的完整示例？
  - [ ] 是否考虑了边缘情况和错误处理？

## 📋 强制执行流程

```
用户请求教学/技术回答
    ↓
触发红灯检查
    ↓
是否全部通过？
    ↓
    否 → 立即停止，先执行验证
    ↓
    是 → 继续回答
    ↓
回答后标注来源
    ↓
更新 Wiki（如有新知识）
```

## 🔴 违反后果

如果违反红灯检查项：
1. **立即停止回答**
2. **执行验证**：读取相关文件
3. **修正回答**：基于真实数据
4. **记录错误**：使用 wiki-capture 记录这次错误
5. **向用户道歉**：承认错误并说明修正措施

## 📊 分层置信度模型

### 三层架构

| 层级 | 定义 | Wiki 有内容 | Wiki 无内容 |
|------|------|------------|-------------|
| **📌 事实层** | 语法、用法、配置项、命令参数 | ✅ 基于 Wiki | ❌ 拒绝 |
| **💡 洞察层** | 场景联想，知识综合、类比 | ✅ Wiki+AI | ⚠️ 标注未验证 |
| **🔮 推测层** | 预测未来、未经证实 | ✅ 谨慎推测 | ⚠️ 加强警告 |

### 标注模板

**洞察层**：
```markdown
> [!tip] AI 洞察
> 💡 此分析基于对 [[page]] 的综合理解，仅供参考。
```

**推测层**：
```markdown
> [!warning] 推测内容
> ⚠️ 此为基于趋势的推测，官方尚未确认。
```

**未验证内容**：
```markdown
> [!caution] 未经验证
> ⚠️ 此内容基于训练数据，Wiki 中暂无相关记录。
```

## 📝 使用方式

每次教学前，内部运行此检查清单：

```
【内部检查】
准备提供教学/技术回答
✓ 检查项 1：Wiki-First 验证
✓ 检查项 2：事实来源验证
✓ 检查项 3：实际示例验证

全部通过 → 继续回答
```

---

**重要**：这不是建议，这是**强制规则**。
违反此规则 = 严重错误 = 必须记录和改进
