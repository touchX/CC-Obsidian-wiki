# 实时监控清单 - 技术教学场景

> **使用方式**：每次准备提供技术教学或回答时，先运行此清单
> **执行时间**：回答之前，约 30 秒

## 🚦 准备回答前检查

### 第一步：快速自检（5秒）

```
问题 1：我要提供的是什么类型的内容？
[ ] 技术教学
[ ] 代码示例
[ ] 概念解释
[ ] 工具使用指导

问题 2：这个内容是基于什么来源？
[ ] 项目 Wiki
[ ] 项目实际文件
[ ] 训练数据（未验证）
[ ] 外部文档

⚠️ 如果是"训练数据（未验证）" → 必须先执行验证
```

### 第二步：来源验证（15秒）

```
[ ] 1. 我读过 wiki/index.md 了吗？
    如果没有 → 立即读取

[ ] 2. Wiki 中有相关页面吗？
    如果有 → 读取并参考
    如果没有 → 标记为"基于训练数据"

[ ] 3. 项目中有实际示例吗？
    如果有 → 读取并验证语法
    如果没有 → 明确说明"这是示例，非实际文件"

✅ 全部通过 → 继续第三步
❌ 任何一项不通过 → 先执行验证
```

### 第三步：一致性检查（10秒）

```
[ ] 我的教学内容与 Wiki 一致吗？
[ ] 我的语法与实际文件一致吗？
[ ] 我标注了来源引用吗？

✅ 全部通过 → 可以回答
❌ 任何一项不通过 → 修正后再回答
```

## 📋 快速决策树

```
开始
  ↓
是技术内容吗？
  ↓ 是
  ↓
读过 Wiki 了吗？ → 否 → 读取 Wiki → 重新开始
  ↓ 是
  ↓
有实际示例吗？ → 否 → 标记"未验证" → 继续
  ↓ 是
  ↓
读取实际示例了吗？ → 否 → 读取示例 → 重新开始
  ↓ 是
  ↓
与来源一致吗？ → 否 → 修正内容 → 重新开始
  ↓ 是
  ↓
标注来源了吗？ → 否 → 添加引用 → 重新开始
  ↓ 是
  ↓
✅ 可以回答
```

## 🎯 强制停止条件

**如果出现以下情况，立即停止回答**：

1. ❌ 发现自己没有读过 Wiki
2. ❌ 不确定内容是否正确
3. ❌ 无法引用具体来源
4. ❌ 项目实际文件与我的理解冲突

**停止后的正确流程**：
1. 承认不确定："我不确定，让我验证一下"
2. 执行验证：读取相关文件
3. 基于真实数据回答
4. 标注来源："来源：xxx 文件第 y 行"

## 📊 自我评分标准

每次回答后给自己打分：

| 评分 | 标准 |
|------|------|
| **A** | 所有验证通过，有明确来源，与实际一致 |
| **B** | 基本验证通过，但来源标注不完整 |
| **C** | 部分验证，有些内容基于训练数据 |
| **D** | 没有验证，凭记忆回答 |
| **F** | 发现错误，需要更正 |

**目标**：每次回答都要达到 **A** 级

---

**核心原则**：30秒检查 > 10分钟错误修正 > 永久信誉损失
