# 技术内容三重验证协议

> **目的**：防止幻觉内容，确保技术准确性
> **适用范围**：所有教学、技术回答、代码示例

## 🔍 三重验证流程

### 第一重：Wiki 验证（必须）

```bash
# 1. 查询 Wiki 索引
Read wiki/index.md

# 2. 找到相关页面
# 例如：Commands 相关 → wiki/guides/commands.md

# 3. 读取相关页面
Read wiki/guides/commands.md
```

**验证问题**：
- ✅ Wiki 中是否有相关内容？
- ✅ Wiki 内容是否准确？
- ✅ 我的教学是否与 Wiki 一致？

### 第二重：实际示例验证（必须）

```bash
# 读取项目中的真实文件
Read .claude/commands/weather-orchestrator.md
Read .claude/agents/planner.md
```

**验证问题**：
- ✅ 实际文件是否支持我的说法？
- ✅ 语法/格式是否完全一致？
- ✅ 我是否遗漏了关键细节？

### 第三重：官方文档验证（推荐）

```bash
# 如果有外部文档链接
WebFetch https://code.claude.com/docs/en/commands
```

**验证问题**：
- ✅ 官方文档是否与我的理解一致？
- ✅ 是否有版本差异需要注意？

## ⚠️ 验证失败处理

| 情况 | 处理方式 |
|------|----------|
| Wiki 无相关内容 | 说明"基于训练数据，建议验证" |
| 实际文件与 Wiki 冲突 | 以实际文件为准，更新 Wiki |
| 无法确定正确性 | 明确说"我不确定，需要验证" |
| 发现 Wiki 错误 | 立即标记并建议修正 |

## 📋 验证检查清单（快速版）

每次教学前快速自检：

```
[ ] 1. 我读过 wiki/index.md 了吗？
[ ] 2. 我读过相关 Wiki 页面了吗？
[ ] 3. 我读过项目中的实际示例了吗？
[ ] 4. 我的教学内容与这些来源一致吗？
[ ] 5. 我能引用具体的文件路径和行号吗？
```

**如果任何一项是"否"** → 立即停止 → 执行验证 → 然后再回答

---

**核心原则**：没有验证 = 不回答 = 防止幻觉
