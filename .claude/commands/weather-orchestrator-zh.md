# 天气编排器命令 (Weather Orchestrator)

获取阿联酋迪拜的当前温度，并生成可视化的 SVG 天气卡片。

## 执行契约（不可协商）

**必须**通过调用 `weather-agent` 子代理来完成此命令。**禁止**：

- ❌ 通过 Bash、WebFetch 或任何其他工具自行获取天气数据
- ❌ 跳过第 1 步（用户温度单位偏好是必需的输入）
- ❌ 在代理返回温度之前调用 `weather-svg-creator`

如果无法调用 Agent 工具，请停止并向用户报告错误。**不要临时发挥**。

### 步骤 1：询问用户偏好

使用 `AskUserQuestion` 工具询问用户希望使用摄氏度还是华氏度。在获得用户选择的单位之前，**不要**继续下一步。

### 步骤 2：通过代理获取天气数据

使用 `Agent` 工具调用天气代理：

| 参数 | 值 |
|------|-----|
| subagent_type | `weather-agent` |
| description | `获取迪拜天气数据` |
| model | `haiku` |
| prompt | 获取阿联酋迪拜的当前温度，单位为用户请求的 **[用户选择的单位]**。返回数值温度和单位。代理有预加载的技能（weather-fetcher）提供详细说明。 |

等待代理完成，并捕获返回的温度数值和单位。

**失败-关闭保护**：如果代理未返回数值温度和单位，**不要**继续第 3 步。向用户报告失败并停止。

### 步骤 3：创建 SVG 天气卡片

使用 `Skill` 工具调用 `weather-svg-creator` 技能：

- skill: `weather-svg-creator`

该技能将使用第 2 步中的温度值和单位（可在当前上下文访问）来创建 SVG 卡片并写入输出文件。

## 输出摘要

向用户提供清晰的摘要，包括：

| 项目 | 值 |
|------|-----|
| 请求的温度单位 | 摄氏度 / 华氏度 |
| 获取的迪拜温度 | XX°C / XX°F |
| SVG 卡片位置 | `orchestration-workflow/weather.svg` |
| 摘要文件位置 | `orchestration-workflow/output.md` |

## 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 用户取消 | 报告用户取消并优雅退出 |
| Agent 超时 | 报告超时错误并停止 |
| Agent 无有效返回 | 报告失败并停止（失败-关闭） |
| SVG 创建失败 | 报告错误并提供温度信息 |
